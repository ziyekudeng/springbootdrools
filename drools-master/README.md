
### 导语
# spring-boot-drools7
> spring-boot整合drools7，数据库实现drools规则文件的增删改查及加载  
> Drools默认加载 *.drl 文件,目前能想到的有两种方案加载规则信息  
> * 1.数据库实现规则的增删改查,然后拼装规则信息,交给规则引擎编译  
     优点:相对灵活,后期运维方便  
     缺点:组装稍显麻烦,尤其逻辑复杂度高的规则  
> * 2.指定规则文件位置加载,例如: FTP 或 MongoDB  
     优点:规则编写容易  
     缺点:运维成本高,需要运维人员会Drools语法
